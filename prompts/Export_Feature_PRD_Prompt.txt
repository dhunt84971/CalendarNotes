Make a PRD file that describes the following changes to be made to the CalendarNotes application.

Add an Export feature for both Documents and Pages.  This feature will be accessed from the context menu for both Documents and Pages.  

When the export feature is selected a modal window will be displayed providing the user with several options for the export.  The export can be to a docx or pdf file and can be in the portrait or landscape format.  It can optionally include a Table of Contents that will use a maximum of 3 levels.  The export will include footer page numbering.  The export will include tabular data in tables with columns that will be aligned based on the markdown specified in the source.  Table column widths will be adjusted to minimize the number of rows. The user can specify the file name and location for the exported file.  By default, the file shall be named based on the name of the page or document being exported.  By default, the file location shall be either the last location from the last export or the user's Documents folder if the last location does not exist or is invalid.  The last location, file type (docx or pdf), table of contents and page orientation (landscape or portrait) shall be saved and used as the starting settings the next time the export feature is accessed.  These options will be saved in the .settings file.

When a single page is exported, markdown content shall be respected when exporting the text to a docx or pdf file.  A Title shall be included using the name of the page.

When a document is exported, the user shall have the option of exporting all of the pages to a single document or to individual documents.  If the user selects to export all of the pages to individual documents, the user shall only be able to specify the location of the documents.  The files shall be named based on the page names.  Page names shall be scrubbed to ensure they create valid filenames for the platform.  If the user selects to export all of the pages to a single document, the default filename will be the name of the document and the page names shall become header 1 level headers.  Markdown headers within the page contents shall be demoted by one within the single exported document.

If the document being exported contains documents, the user shall have the option of exporting all of the documents and their pages below the level of the selected document provided the user has selected to export all pages to a single document.  This option shall be called Export all sub documents.  When selected, the user will no longer be able to specify the filename for the export.  They will only be able to specify the location.  In addition, the export feature will recursively export each sub document's pages to a single file in the specified export location to a file named for the sub document.  Sub document names are to include the name of the parent document in the format <parent name> - <sub name>.
